1. 项目简介
本仓库为 GitHub 小组协作实训项目，核心采用番茄钟 25min 专注 + 5min 休息工作法管控全流程，标准化完成仓库搭建、任务拆解、分支开发、PR 评审合并全链路，实现高效小组增量协作开发。

2. 实训核心目标
搭建独立小组 GitHub 仓库，使用番茄钟拆分全部开发任务
每位成员按番茄单元完成至少 1 次规范 Commit 提交
小组完整走完一轮 PR 发起、评审、修改、合并流程
全程保障main分支稳定，所有工作以番茄单元量化进度

3. 前置环境依赖
本地安装 Git，配置用户名与绑定 GitHub 账号
番茄计时工具（手机计时器 / 番茄 ToDo / 专注森林均可）
代码编辑器（VS Code 推荐）

5. 全流程番茄钟拆分执行方案（核心内容）
标准规则：25 分钟专注单一任务，中途不切换工作；完成 1 个番茄后休息 5 分钟；累计 4 个番茄进行 20 分钟长休息。

番茄 1（25min）｜仓库初始化
工作内容：新建 GitHub 仓库、添加组员、配置分支保护、初始化远程目录
休息 5min：核对仓库链接、成员权限全部到位，记录番茄完成标记

番茄 2（25min）｜整理项目文档（含本 README）
工作内容：编写 README、搭建文件夹结构、撰写 docs 内协作规范
休息 5min：组员通读文档，收集修改意见，记录待优化点

番茄 3（25min）｜创建 Issue + 分配任务
工作内容：拆分全流程工作，批量创建 Issue，组员认领对应任务，填写tomato-plan.md番茄计划表
休息 5min：核对任务无重复、无遗漏，确认每人分配番茄工作量均衡
累计 3 个番茄，等待下一轮完成第 4 个后长休

番茄 4（25min）｜创建个人开发分支
工作内容：本地克隆仓库、基于main创建个人功能分支、本地环境调试
休息 5min：统一记录 4 轮番茄完成
✅ 长休息 20min：小组集中沟通前期流程卡点、调整后续番茄任务量

番茄 5（25min）｜代码开发 + 规范 Commit 提交
工作内容：完成认领 Issue 任务，分段编写代码，规范提交 commit 并推送远程分支
休息 5min：检查远程分支同步状态，核对 commit 格式是否合规

番茄 6（25min）｜发起 Pull Request
工作内容：关联对应 Issue 创建 PR，填写修改说明、自测结果，@评审组员
休息 5min：确认所有评审人员收到通知，记录 PR 编号至番茄计划表

番茄 7（25min）｜代码 Review + 问题修改
工作内容：交叉评审代码，标注修改意见；开发人员在分支修复问题，重新推送更新 PR
休息 5min：确认所有评审意见处理完毕

番茄 8（25min）｜合并代码 + 整体复盘验收
工作内容：评审通过后合并至main、清理废弃分支；对照实训目标、番茄计划表完成全部交付物核对
休息 5min：整理本次协作番茄执行问题，更新优化计划表

7. 提交规范 & 番茄记录规范
Commit 统一格式：[番茄X-分类] 修改描述
示例：[番茄2-文档] 新增番茄钟执行流程章节
每完成 1 个番茄，在tomato-plan.md内打勾记录，标注卡点与耗时问题
中断作废规则：番茄 25 分钟内中途切换任务、频繁走神，本番茄作废，休息后重新计时

8. FAQ
Q1：一个番茄内任务做不完怎么办？
A1：将任务拆分为更小单元，新增对应 Issue 分配至下一轮番茄，禁止跨番茄堆积任务。
Q2：小组多人番茄计时不同步如何处理？
A2：小组统一开启计时器，同步进入 5 分钟休息时段，休息时统一沟通协作问题。
Q3：代码冲突、PR 报错占用大量专注时间？
A3：中断当前番茄，5 分钟休息梳理解决方案，下一轮番茄专门处理故障修复。

main 分支稳定，文档清晰记录每轮番茄工作成果与问题复盘
